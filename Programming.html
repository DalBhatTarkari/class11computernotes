Low Level Language

Low-level languages provide no abstraction over computer hardware and allow direct control over the computer's resources.

Characteristics:

  • Provides direct control over the computer's hardware and resources.
  • Program written is machine-dependent, making it non-portable and difficult to write.
  • Programs written in low-level language run faster.

Types

Machine Language (1GL)

  • Consists of binary numbers (1 and 0).
  • Directly understood by the processor.
  • No language translator is required.
  • Faster execution speed and efficient use of hardware.
  • Difficult to develop and debug programs.
  • Program written is machine-dependent.
  • Requires a deeper understanding of hardware architecture.
  • Operation codes and memory addresses must be remembered.
  • Highly prone to errors.

Assembly Language (2GL)

  • Uses symbolic instructions like ADD, SUB, DIV, etc.
  • Assembler converts assembly code to machine code (source code to object code).
  • Easier to develop and debug programs compared to machine language.
  • Faster execution of code and efficient use of hardware components than high-level languages.
  • Useful for developing hardware-oriented software like device drivers and robotics.
  • Not directly understood by the computer; needs an assembler for translation.
  • Program written is machine-dependent.
  • Requires a deeper understanding of hardware architecture.
  • Highly prone to errors.


Operators

Operators are symbols used to perform operations on operands (values or variables).

  • Operand: A quantity on which an operation is performed (e.g., in 7 - 2, both 7 and 2 are operands).
  • Operator: A symbol that represents an operation (e.g., in 7 - 2, "-" is the operator indicating subtraction).

Types of Operators:

  • Arithmetic Operators: +, -, *, /
  • Relational Operators: >, <, >=, <=
  • Logical Operators: AND, OR, NOT

Operation: The process of applying an operator to operands (e.g., in 7 - 2, the operation is subtraction, resulting in 5).



Errors

An error is an issue in the code or execution process that prevents a program from functioning as intended.

Types of Errors

  • Syntax Error: When the code doesn't follow the syntax or writing format of the respective language.
    Example: Missing semi-colon at the end of a line in C, missing {} in a for loop in JavaScript, etc.
  • Logical Error: An error in the program's logic or concept that causes it to operate incorrectly but not terminate abnormally.
    Example: Using area = length + breadth instead of length * breadth.
  • Semantic Error: Semantic errors happen when the code is syntactically correct but does not work as intended
    Example: using x = y * z instead of x = y + z when summation is needed
  • Runtime Error: An error that occurs while the program is executing or running.
    Example: Accessing invalid memory or causing a memory leak.
  • Compilation Error: Issues detected during the translation of source code to machine code.
    Example: Using undeclared variables, not importing required libraries, etc.

Syntax

The set of rules that defines the structure of a programming language. Syntax errors prevent the code from running.

Semantics

The meaning or behavior of the code, i.e., what the code does when executed. Semantic errors cause incorrect program output.It is similar to gramatical error in a human language

Testing

The process of evaluating a program to ensure it functions as expected. It involves identifying bugs, errors, or missing requirements before deployment.

Debugghing

The process of finding and fixing the bugs or errors identified during testing or program execution.

Types of Testing

  • Unit Testing: Testing individual components. Example: Testing a `calculate_sum()` function in isolation.
  • Integration Testing: Testing the interaction between different components. Example: Checking if a payment module works with a shopping cart system.
  • System Testing: Testing the entire system as a whole. Example: Verifying a complete e-commerce website, from user login to payment processing.

Program Design Tools

Algorithm

An algorithm is a finite and step-by-step set of statements that solve a particular problem.

Pseudocode

Pseudocode is a program design tool that uses English phrases and mathematical statements to explain program flow.

Flowchart

A flowchart is a diagrammatic representation of a program flow that represents a sequence of operations to be followed in a particular order.
